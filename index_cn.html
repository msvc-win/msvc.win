msvc.win

只需一条命令即可安装 Visual C++、DirectX 和 .NET Framework 3.5！

快速指南

您只需以下三个简单的步骤即可安装 Visual C++、DirectX 和 .NET Framework 3.5 运行库。

  1. 打开 PowerShell
    请按下键盘的 Windows 键 + X (或右键点击“开始”按钮)，然后选择 PowerShell 或 终端。

  2. 复制并粘贴以下代码(按 Ctrl + V 或右键单击窗口)，然后按回车键。 

    irm https://get.msvc.win | iex

  3. 如提示 UAC 提示，请点击确认，然后等待完成即可。建议您在继续操作前，禁用或卸载第三方反病毒软件 (尤其是360安全卫士)。

常见问题

  1. 为何创建这个网站？
    有些在互联网上分发的程序或游戏通常没有捆绑必要的运行时环境，用户可能会因此遇到很多麻烦。 我见过很多玩家对此毫不知情，他们不断地卸载并重新安装那些没有捆绑运行时环境的游戏，希望最终能解决问题。
    如果您经常重装 Windows 系统，那么除了驱动程序之外，这绝对是您最想安装的东西。 我想要一个简单易用的解决方案，而不是误点到广告软件。
    此外，这个域名很容易记住：msvc 代表 Microsoft Visual C++，而 win 代表 Windows。

  2. 使用起来安全吗？
    是的。我们绝不会对相关脚本其进行混淆处理，因此脚本内容易于理解。 它使用的唯一第三方程序是aria2c，这是一个开源的 基于命令行的下载器，用于更快地下载必要的安装程序。
    所有必要的安装程序均直接从微软服务器下载，以确保您始终获得正规且最新的运行时环境。
    欢迎您在我们的 GitHub 代码库中查看和贡献代码。

  3. 我的电脑上 Visual C++ 运行库安装损坏，或是由于缺少 MSI 文件而无法卸载。我该如何修复？
    你可以下载 Windows Installer CleanUp Utility 7.2来强制删除损坏的运行库安装。如需操作：

    • 请右键单击下载的 msicuu2.exe 文件，选择“以管理员身份运行”，以管理员权限运行它。如果您看到与 VBScript 相关的错误消息，则表示您没有以管理员身份运行它。
    • 在“开始”菜单中找到“Windows Installer CleanUp Utility”，右键单击它，然后选择“更多” - “以管理员身份运行”。
    • 选择所有以“Microsoft Visual C++”开头的条目，然后单击“删除”，并确认删除。如果您看不到这些条目或无法删除它们，则表示您没有以管理员身份运行该程序。
    • N现在您可以尝试再次执行安装运行库的命令。

  4. 我可以省略 https:// 前缀吗？
    不建议这样做，但您可以省略。
    这是为了防止脚本可能被恶意网络服务提供商劫持的预防措施。由于部分用户可能不知道如何 正确输入冒号和竖线符号，因此我们允许您省略 https:// 前缀。

  5. 与大多数离线软件包(例如 DirectX Repair、abbodi1406 的 VisualCppRedist_AIO)相比，该脚本的优缺点是什么？
    优点：

    • 你只需要记住一个命令，而不用从任何地方查找软件包下载。
    • 该脚本是开源的，并且始终会从微软官方网站下载正版文件，而不是点击搜索引擎上的恶意付费搜索结果。
    • 该脚本始终会将您引导至最新稳定版本的 Visual C++ 运行库安装包。
    缺点：
    • 需要互联网连接。
    • 不支持未安装最新更新的过时版本 Windows (例如 Windows 7)，也不鼓励使用。

  6. 执行前是否需要卸载已安装的 Visual C++ 运行库？
    不。安装程序会自动将现有运行时更新到最新版本。 例如，如果您已经安装了 Visual C++ 2015 Redistributable Runtime(初始版本)， 它将自动更新到最新版本。

  7. 我正在使用最新的 Windows 11 内部预览版本。我仍然可以安装 .NET Framework 3.5 吗？
    是的，它会自动下载适用于 Windows 11 Insider Release 的合适版本(称为 .NET Framework 3.5 Preview for Windows vNext)，因为从 Build 27965 开始，之前使用 Windows 安装 ISO 安装 .NET Framework 3.5 的方法不再有效。

  8. 是否有只安装 32 位 Visual C++ 运行库的选项？
    是的。您可以执行此命令来安装 32 位 Visual C++ 运行库，而无需安装 DirectX： 

    irm https://get.msvc.win/32 | iex
    同样，您也可以执行此命令来安装 32 位和 64 位 Visual C++ 可再发行组件运行时，而无需安装 DirectX： 
    irm https://get.msvc.win/64 | iex
    也可以执行此命令仅安装 DirectX： 
    irm https://get.msvc.win/dx | iex
    还可执行此命令仅安装 .NET Framework 3.5： 
    irm https://get.msvc.win/net35 | iex

  9. 是否支持 Windows 7 / Server 2008 R2？
    可以，但需满足以下条件：

    • 需要安装最新的安全更新以及 Service Pack 1 (如果还未安装), 以验证微软的 SHA-2 签名。 为了节省精力，我们建议使用 simplix 的 UpdatePack7R2 完成更新。
    • 安装 .NET Framework 4.8。
    • 根据你 Windows 的架构来安装 Windows Management Framework 5.1。具体而言，你需要安装里面的 KB3191566 msu 安装包。

    然后你就可以使用相关命令了。还是忘了 XP 和 Vista 吧。

  10. 我可以通过代理下载吗？
    可以。如果您配置了系统代理，它将通过代理自动下载必要的文件。 针对中国用户在通过 GitHub 发布版本下载文件时遇到的困难，或 ISP 提供的 DNS 解析结果存在污染等问题，我们将从 Gitee 下载 aria2c 二进制文件，并自动使用 AliDNS (233.5.5.5) 从 Microsoft 服务器下载安装程序。 系统代理配置的优先级高于 AliDNS。

  11. 我想将其整合进 Windows 安装镜像，要如何操作？
    你可以在这里下载整合包。
    您可以将解压的 $OEM$ 文件夹和 autounattend.xml 文件放入 Windows 安装映像的 sources 文件夹，或者放入由 Windows 媒体创建工具创建的 USB 驱动器的 sources 文件夹中。
    如果您想将脚本集成到 install.wim 中，请将 autounattend.xml 重命名为 unattend.xml 并将其放入 install.wim 的 Windows/Panther 文件夹（如果不存在则创建一个），然后将 GetMSVC.ps1 从 $OEM$/$$/Setup/Scr​​ipts 复制到 install.wim 的 Windows/Setup/Scr​​ipts 文件夹（如果不存在则创建一个）。
